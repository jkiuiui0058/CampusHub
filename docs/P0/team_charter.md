# 团队章程

## 团队成员与角色分工


| 角色 | 姓名 | 职责 | 后续主导阶段 |
|------|------|------|-------------|
| 需求负责人 | 于静涵 | 负责需求获取、文档撰写、用例建模 | P1 需求分析 |
| 架构负责人 | 马志远 | 负责技术选型、架构设计、ADR 撰写 | P2 架构设计 |
| 开发负责人 | 陈婉清 | 负责编码规范、代码审查、核心模块开发 | P4 核心编码 |
| 测试负责人 | 戴安信 | 负责测试策略、测试用例设计、质量保障 | P6 质量保障 |

注意：
- 负责人负责主导阶段任务，如组织讨论、分配任务、汇总产出，最终把关。
- 按照负责人安排，所有环节均需全员参与
