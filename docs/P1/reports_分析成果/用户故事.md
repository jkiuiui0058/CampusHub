# 用户故事、验收标准及优先级排序

### A. 需求发布者（Requester）

**US-01（Must）**  
作为需求发布者，我希望按类型发布任务（跑腿/失物招领/二手/学习求助等），以便快速描述清楚需求。

验收标准：
1. Given 我已登录并进入发布页  
   When 我选择任务类型  
   Then 系统展示该类型对应的结构化字段（地点、时间、酬劳、说明等）。

**US-02（Must）**  
作为需求发布者，我希望简单任务可一键快速发布，以便在紧急情况下减少操作步骤。

验收标准：
1. Given 我发布“快递代取/带物”等简单任务  
   When 我仅填写必填项并提交  
   Then 任务在 3 步内完成发布并进入待接单状态。

**US-03（Must）**  
作为需求发布者，我希望复杂任务支持详细描述和分阶段确认，以便降低“做不清楚”的风险。

验收标准：
1. Given 我发布复杂任务  
   When 我填写阶段目标与验收方式  
   Then 任务详情页可按阶段记录完成状态。

**US-04（Must）**  
作为需求发布者，我希望对外仅展示必要信息（如手机尾号），以便保护隐私。

验收标准：
1. Given 我在发布页设置“联系方式可见范围”  
   When 任务被他人浏览  
   Then 页面只展示我勾选的公开字段，真实姓名/学号/完整手机号默认不公开。

**US-05（Should）**  
作为需求发布者，我希望编辑未被接单的任务，以便修正描述错误。

验收标准：
1. Given 任务状态为“待接单”  
   When 我修改地点或时间并保存  
   Then 列表与详情页实时显示最新内容并保留更新时间。

**US-06（Must）**  
作为需求发布者，我希望可以取消未完成任务，以便应对需求变化。

验收标准：
1. Given 任务状态为“待接单/进行中”且满足取消规则  
   When 我点击取消并确认原因  
   Then 任务状态变为“已取消”，并通知相关接单人。

**US-07（Could）**  
作为需求发布者，我希望紧急任务获得更高曝光，以便更快匹配到帮助者。

验收标准：
1. Given 我发布任务并标记“紧急”  
   When 其他用户浏览任务流  
   Then 该任务在同类型列表中优先展示，并带有“紧急”标识。

**US-08（Should）**  
作为需求发布者，我希望任务完成后可评价帮助者，以便形成长期信用记录。

验收标准：
1. Given 任务状态为“待确认完成”  
   When 我确认完成并提交评分与评价  
   Then 对方信用页新增一条可追溯评价记录。

### B. 任务执行者（Helper）

**US-09（Must）**  
作为任务执行者，我希望按类型/关键词/标签搜索任务，以便快速找到能做的单子。

验收标准：
1. Given 我在任务大厅  
   When 我按“二手/跑腿/失物招领”等筛选并输入关键词  
   Then 结果仅返回匹配任务，并可按发布时间/紧急度排序。

**US-10（Must）**  
作为任务执行者，我希望在接单前查看发布者信用与历史评价，以便判断风险。

验收标准：
1. Given 我打开任务详情  
   When 我查看发布者信息  
   Then 可看到认证状态、历史完成率、评分与近期评价摘要。

**US-11（Must）**  
作为任务执行者，我希望接单后必须使用站内聊天沟通，以便关键约定可留痕。

验收标准：
1. Given 我已接单  
   When 我尝试联系发布者  
   Then 系统自动进入站内会话并保存文字/图片记录。

**US-12（Must）**  
作为任务执行者，我希望提交完成凭证（照片/截图/单号），以便证明已履约。

验收标准：
1. Given 任务接近完成  
   When 我点击“提交完成”并上传凭证  
   Then 任务状态变为“待发布者确认”，且凭证可被双方与管理员查看。

**US-13（Should）**  
作为任务执行者，我希望同样能评价发布者，以便约束“恶意压价/拒不确认”等行为。

验收标准：
1. Given 任务已完成或已关闭  
   When 我提交对发布者的评分  
   Then 发布者信用页新增对应评价条目。

**US-14（Should）**  
作为任务执行者，我希望看到明确的酬劳形式（现金/积分等），以便判断是否接单。

验收标准：
1. Given 我浏览任务详情  
   When 查看酬劳信息  
   Then 可看到酬劳类型、数额与结算说明。

### C. 平台管理员（Admin）

**US-15（Must）**  
作为平台管理员，我希望通过校内身份认证准入机制拦截非本校用户，以便提升基础信任。

验收标准：
1. Given 新用户注册或登录  
   When 用户未完成校内身份核验  
   Then 系统限制其发布/接单能力，仅允许完成认证流程。

**US-16（Should）**  
作为平台管理员，我希望处理纠纷并调取证据，以便对“是否完成任务”做出裁定。

验收标准：
1. Given 任一方发起申诉  
   When 管理员进入工单详情  
   Then 可查看站内聊天记录、完成凭证与时间线，并输出处理结果。

**US-17（Could）**  
作为平台管理员，我希望对违规账号采取分级处罚（警告/限权/封禁），以便降低诈骗与失约。

验收标准：
1. Given 用户被多次投诉且证据成立  
   When 管理员执行处罚  
   Then 系统记录处罚原因、时长与申诉入口，并即时生效。

**US-18（Could）**  
作为平台管理员，我希望上线“社区小法庭”投票仲裁辅助，以便提升处理透明度与公信力。

验收标准：
1. Given 纠纷进入可投票状态  
   When 平台邀请高信用用户参与裁决  
   Then 系统汇总投票意见供管理员最终判定参考。

**US-19（Should）**  
作为平台管理员，我希望对长期未响应任务做自动重推或下架，以便避免信息被淹没。

验收标准：
1. Given 任务在设定时间内无人接单或无互动  
   When 到达重推/过期阈值  
   Then 系统自动执行重推或下架，并通知发布者。
