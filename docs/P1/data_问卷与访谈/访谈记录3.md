1.**你最近一次在校园里需要别人帮忙，或者帮别人做事，是什么情况？最后是怎么解决的？** 最近一次需要别人帮忙的情况是找人代课并帮忙拍PPT，最后是通过找认识的人来解决这个需求的。

2.**当时你为什么选择那个渠道来解决，而不是别的渠道？** 选择找认识的同学帮忙，是因为同学之间可以互帮互助，而且这种帮忙是免费的。

3.**整个过程中最麻烦、最容易失败、或者最让你不放心的一步是什么？** 整个过程中最让人不放心的是两方面，一方面是怕代课的同学睡过头耽误事，另一方面是怕代课的时候遇到老师查手机这类特殊情况。

4.**如果把你刚才那次经历放到一个校内互助平台上，你觉得平台最应该先帮你解决什么问题？** 如果放到校内互助平台上，平台要能保证接单帮忙的人确实会认真完成任务。

5.**你觉得这些不同类型的需求，发布和完成流程应该一样吗？哪里应该不一样？** 不同类型的需求，发布和完成流程不应该一样。差异主要在于个人信息的提供上，有些需求需要提供个人信息，有些需求则不需要提供个人信息。

6.**如果你要发布一个互助需求，你最希望填写哪些信息？哪些信息你不愿意公开？** 发布互助需求时，只愿意填写需求必需的信息，比如取外卖只需要提供手机尾号，代课只需要提供姓名；除此之外的多余信息，都不愿意公开。

7.**你怎么看“校内身份认证”？你觉得哪种认证方式最合适，哪种会让你犹豫？** 认可校内身份认证的方式，觉得用学号这类学校相关信息进行认证是最合适的；如果需要输入身份证信息进行认证，会让人犹豫，觉得不太靠谱。

8.**什么样的信息会让你更愿意相信对方，比如认证、历史评价、信用分、学院/校区信息？** 更愿意相信能看到对方积分、历史做事记录以及好评情况的信息，就像在闲鱼平台上能看到卖家的评价和粉丝数一样，这类信息会增加信任感。

9.**你最担心平台上出现什么问题？如果真发生了，你希望平台怎么处理？** 最担心的问题是追责困难，花钱找人办事但对方没干好的话，双方容易产生纠纷，而且平台信息不够透明。如果发生这类问题，希望平台一方面保证用户信息全面且做好保密工作，同时拥有一定的执行权；另一方面可以设置类似闲鱼小法庭的仲裁机制来处理纠纷。

10.**如果你帮助了别人，你理想中的回报是什么？如果只能有一种激励方式，为什么你能接受或不能接受积分？** 帮助别人后，理想中的回报是现金。如果只能选择一种激励方式，也可以接受积分，只是觉得积分不太方便，因为如果积分不能提现的话，就只能在平台内使用。

11.**如果平台收费，你觉得什么情况下可以接受？什么情况下绝对不能接受？** 可以接受平台适当收费，毕竟可以支持平台的开发和维护；但如果平台收费过高，就绝对不能接受。

12.**在任务进行过程中，你觉得站内聊天是必须的吗？哪些沟通内容一定要留痕？** 任务进行过程中，站内聊天是必须的。需要留痕的沟通内容是任务相关的关键信息，包括任务执行的地点、方式、具体内容，以及需要特别注意的事项，这些内容要明确双方的权利和义务。

13.**你觉得一个任务“完成了”应该由谁来确认？需要什么证据？** 任务“完成了”应该由需求方和帮忙方双方达成合意来确认；同时双方可以提供照片等材料作为完成任务的证据，平台仅作为保底的保障手段。

14.**如果双方对“有没有完成任务”意见不一致，你觉得平台应该怎么判断？** 如果双方对任务是否完成意见不一致，首先可以由平台客服介入，组织双方进行调解沟通；如果调解无效，就可以启动类似小法庭的机制，依据之前留痕的沟通记录等内容进行投票仲裁。
